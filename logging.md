# 📝 백엔드 API 감사 로그(Audit Logging) 가이드

이 문서스크립트는 서비스 운영 및 보안 감사를 위해 구축한 **5W1H 기반의 백엔드 API 감사 로그 시스템**의 구조와 형식, 확인 방법에 대해 설명합니다.

---

## 1. 개요 및 목적
- **PM2 모니터링 연동**: Next.js 프로덕션 환경(`next start`)에서는 200 OK 등의 성공 트래픽 로그가 기본적으로 남지 않습니다. 이를 극복하고 실시간 운영 상황 및 어뷰징 시도를 모니터링하기 위해 명시적인 콘솔 출력을 구현했습니다.
- **보안 감사 및 장애 분석**: 비정상적인 입력 요청이나 접근 권한 침해 시도를 타임스탬프 및 출발지 정보와 함께 영구히 PM2 로그에 기록함으로써 보안 침해 사고 및 예외 장애 원인 파악을 용이하게 합니다.

---

## 2. 5W1H 로그 구성 표준
익명성 보장이라는 서비스 철학에 따라, 개인 식별 정보(닉네임, 비밀번호 등)는 기록에서 제외하고 **클라이언트 IP 주소**를 핵심 식별 정보로 대체하여 아래와 같이 5W1H 구조로 포맷을 일원화했습니다.

| 구성 요소 | 정의 | 수집 방식 / 구현 내역 |
| :--- | :--- | :--- |
| **When (언제)** | 요청 처리 시간 | 신뢰할 수 있는 `ISO 8601 UTC Timestamp` (`new Date().toISOString()`) |
| **Who (누가)** | 요청 행위자 식별 | **익명 게시판 특성상 IP 주소로 대체** (로그 내 닉네임/비밀번호 배제) |
| **Where (어디서)** | 출발지 및 기기 식별 | Nginx 프록시를 통과한 실제 공인 IP (`x-forwarded-for`) + 기기 정보 (`User-Agent`) (서버사이드에서 `geoip-lite` 라이브러리를 통해 IP 기준 국가/도시 정보를 분석하여 DB에 `location` 컬럼으로 기록하며 어드민에서 노출됨) |
| **What (무엇을)** | 처리 목적 및 데이터 | 신규 글 작성(POST), 글 수정(PUT), 글 삭제(DELETE), 어드민 로그인 시도 등 |
| **How (어떻게)** | 프로토콜 및 메서드 | HTTP Request Method (`POST`, `PUT`, `DELETE`) + 요청 경로 |
| **Result (결과)** | 최종 성공 여부 | 결과 상태 코드 (`200 OK`, `201 Created`, `400 Bad Request`, `403 Forbidden`, `404 Not Found`) |

---

## 3. 로그 메시지 포맷 예시

### 🟢 1) 글 작성 (POST /api/posts)
- **요청 접수 시 (Request):**
  ```text
  [2026-06-18T07:29:10.123Z] [API LOG] [POST] /api/posts - 새 글 작성 요청 들어옴 (IP: 172.30.1.82, UA: Mozilla/5.0...)
  ```
- **작성 성공 시 (201 Created):**
  ```text
  [2026-06-18T07:29:10.145Z] [API LOG] [201 Created] 글 작성 완료 (IP: 172.30.1.82, 등록된 글 번호: 42, 작성자: 홍길동, 내용 일부: "안녕하세요...")
  ```
- **검증 실패 시 (400 Bad Request):**
  ```text
  [2026-06-18T07:29:11.002Z] [API LOG] [400 Bad Request] 글 작성 실패 - 내용 길이 초과 (IP: 172.30.1.82)
  ```

### 🟡 2) 글 수정 (PUT /api/posts)
- **요청 접수 시 (Request):**
  ```text
  [2026-06-18T07:29:12.456Z] [API LOG] [PUT] /api/posts - 글 수정 요청 들어옴 (IP: 172.30.1.82, 대상 글: 42, UA: Mozilla/5.0...)
  ```
- **비밀번호 불일치 시 (403 Forbidden):**
  ```text
  [2026-06-18T07:29:12.478Z] [API LOG] [403 Forbidden] 글 수정 실패 - 비밀번호 불일치 (IP: 172.30.1.82, 대상 글: 42)
  ```

### 🔵 3) 기타 백엔드 API 로그 표준 포맷
수정이 이루어진 모든 백엔드 엔드포인트도 아래와 같이 5W1H 규격으로 동일하게 로깅됩니다.

* **이미지 업로드 (`POST /api/upload`)**
  * *요청 접수:* `[2026-06-18T07:29:13.123Z] [API LOG] [POST] /api/upload - 이미지 업로드 요청 들어옴 (IP: 172.30.1.82, UA: Mozilla/5.0...)`
  * *업로드 성공:* `[2026-06-18T07:29:13.345Z] [API LOG] [200 OK] 이미지 업로드 성공 (IP: 172.30.1.82, 파일명: 178177...png)`
* **새 게시판 생성 (`POST /api/boards`)**
  * *요청 접수:* `[2026-06-18T07:29:14.123Z] [API LOG] [POST] /api/boards - 새 게시판 생성 요청 들어옴 (IP: 172.30.1.82, UA: Mozilla/5.0...)`
  * *생성 성공:* `[2026-06-18T07:29:14.345Z] [API LOG] [201 Created] 새 게시판 생성 완료 (IP: 172.30.1.82, 보드 ID: 4, 이름: board-1)`
* **글 권한(비밀번호) 검증 (`POST /api/posts/verify`)**
  * *요청 접수:* `[2026-06-18T07:29:15.123Z] [API LOG] [POST] /api/posts/verify - 게시글 권한 검증 요청 들어옴 (IP: 172.30.1.82, 대상 글: 42, UA: Mozilla/5.0...)`
  * *인증 성공:* `[2026-06-18T07:29:15.234Z] [API LOG] [200 OK] 게시글 권한 검증 성공 (IP: 172.30.1.82, 대상 글: 42)`
* **어드민 이미지 및 글 검열 (`DELETE /api/admin/images`)**
  * *요청 접수:* `[2026-06-18T07:29:16.123Z] [API LOG] [DELETE] /api/admin/images - 이미지 검열 처리 시도 (IP: 172.30.1.82, 파일: 178177...png, 액션: delete-post, UA: Mozilla/5.0...)`
  * *검열 완료:* `[2026-06-18T07:29:16.456Z] [API LOG] [200 OK] 이미지 검열 완료 - 게시글 소프트 딜리트 및 이미지 소거 (IP: 172.30.1.82, 파일: 178177...png)`
* **어드민 인증 (`POST /api/admin/verify`)**
  * *인증 시도:* `[2026-06-18T07:29:17.123Z] [API LOG] [POST] /api/admin/verify - 어드민 인증 시도 (IP: 172.30.1.82, UA: Mozilla/5.0...)`
  * *인증 성공:* `[2026-06-18T07:29:17.234Z] [API LOG] [200 OK] 어드민 인증 성공 (IP: 172.30.1.82)`
* **어드민 PM2 로그 조회 (`GET /api/admin/logs`)**
  * *조회 시도:* `[2026-06-18T07:29:18.123Z] [API LOG] [GET] /api/admin/logs - 어드민 PM2 로그 조회 시도 (IP: 172.30.1.82, UA: Mozilla/5.0...)`
  * *조회 성공:* `[2026-06-18T07:29:18.234Z] [API LOG] [200 OK] 어드민 PM2 로그 조회 성공 (IP: 172.30.1.82)`
* **어드민 전체 게시글 목록 조회 (`GET /api/admin/posts`)**
  * *조회 시도:* `[2026-06-18T07:29:19.123Z] [API LOG] [GET] /api/admin/posts - 어드민 전체 게시글 목록 조회 시도 (IP: 172.30.1.82, UA: Mozilla/5.0...)`
  * *조회 성공:* `[2026-06-18T07:29:19.234Z] [API LOG] [200 OK] 어드민 전체 게시글 목록 조회 성공 - 총 100건 (IP: 172.30.1.82)`

### 🔴 4) 예외 에러 (Try-Catch 예외)
모든 백엔드 API의 catch 블록 에러 로깅 형식을 표준화하여, PM2 로그에서 에러 스택 트레이스 필터링이 가능하도록 접두사를 추가했습니다.
```text
[API ERROR] posts API error Error: connect ECONNREFUSED 127.0.0.1:5432
    at ...
```

## 4. 실시간 로그 모니터링 (PM2)
원격 서버 환경에서 API 로그를 실시간 모니터링하려면 아래의 PM2 명령어를 사용합니다.

```bash
# 전체 실시간 로그 스트리밍
pm2 logs map-board

# 정상 요청/성공 로그만 필터링해서 보기
pm2 logs map-board | grep "\[API LOG\]"

# 예외/에러 로그만 필터링해서 보기
pm2 logs map-board | grep "\[API ERROR\]"
```

## 5. 어드민 페이지 실시간 로그 뷰어 연동
관리자 대시보드(`/admin`)의 `실시간 서버 로그` 탭에서 PM2 로그를 확인할 수 있습니다.

### PM2 로그 경로 설정 (`.env`)
사용자 서버 환경에 따라 PM2 로그가 저장되는 경로가 다를 수 있으므로, 아래와 같이 `.env` 파일에 `PM2_USER` 변수를 설정하여 PM2가 실행 중인 사용자 계정을 개별 지정할 수 있습니다.
```env
PM2_USER="pyeongju"
```
설정하지 않을 경우, 기본적으로 현재 Node.js 프로세스를 구동 중인 사용자의 홈 디렉터리(`~/.pm2/logs/`)를 기준으로 로그 파일을 탐색합니다.
